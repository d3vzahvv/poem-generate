The Chinese Poems Generator

Brief Intro: Long Short-time Memory, words vector, Pytorch, torch.nn, flask(front end), hand-write


Details:

The pre-training model (model.pkl) supports x86(amd64) CPU

Dataset and paraments: 300 poems for 1000 epochs / hiddenNum = 128 / batchSize = 16

The whole project has been decoupled (although it's not clear enough)

Workflow:
Preprocess dataset (split poems to every single word) -> Vectorization -> Training -> Generate Model -> Load Model -> Generate Poems

Usage:

Enter 4 Simplified Chinese characters (aka Hanzi / Chinese Kanji) then the model Generates a "4 sentences 7 words traditional Chinese poem", every sentence starts with the entered character

If the number of entered characters aren't 4 (even there's no character) or the entered characters aren't Simplified Chinese characters (like English or other Language), the model will still print a Chinese poem. Because the model uses the word vectors to forecast next character

Due to the small test model (300 poems dataset), the model may generate poems don't start with the entered characters which are never included by dataset
